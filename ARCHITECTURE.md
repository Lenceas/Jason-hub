# ARCHITECTURE.md

> 描述 Jason-hub 项目整体的架构设计、模块划分及各子项目之间的关系。

## 整体架构

```
┌─────────────────────────────────────────────────────┐
│              用户访问 (浏览器)                        │
└────────────┬───────────────────────────┬─────────────┘
             │                           │
             ▼                           ▼
┌─────────────────────┐     ┌───────────────────────┐
│   Portfolio (主站)   │     │   子项目独立页面/站点   │
│   Astro 6 · 静态生成  │ ──→ │   Vue 3 + TypeScript  │
│                      │     │   (独立部署 / 子域名)   │
│  个人介绍 + 项目卡片集  │     │                       │
└─────────────────────┘     └───────────────────────┘
```
- **Portfolio**（Astro 主站）是用户入口，展示个人信息和所有项目卡片
- 点击项目卡片跳转到对应**子项目**的独立站点（子域名）
- 每个子项目可独立开发、独立部署、独立迭代

## 主站（Portfolio）

Astro 6 单页应用，输出纯静态 HTML。

**页面组成：**
| 区块 | 组件 | 说明 |
|------|------|------|
| 顶栏 | Header | Logo + 导航文字 |
| 个人介绍 | Hero | 头像、名称、标语、简介、社交链接（品牌色图标按钮） |
| 技能标签 | Skills | 彩色标签云，按熟练度分色（内联样式） |
| 价值观展示 | Values | 水墨风格程序员价值观 12 词（内联样式，响应式三组并排，自动轮播高亮） |
| 项目卡片 | ProjectGrid → ProjectCard | 从 projects.json 渲染卡片网格（毛玻璃效果） |
| 页脚 | Footer | 版权信息 + 构建说明 |

**数据流：**
```
src/data/projects.json  ──→  Astro 构建时读取  ──→  静态 HTML
         ↑
   添加新子项目时追加一条记录
```

**视觉特效：**
| 特效 | 说明 |
|------|------|
| 鼠标跟随光晕 | Hero 区淡蓝光晕随鼠标移动 |
| 渐变装饰圆 | Hero 区背景半透明渐变圆 |
| 碰撞弹球 | 6 色半透明小球在页面内弹跳碰撞 |
| 滚动动画 | 元素进入视口时淡入上浮 |
| 毛玻璃卡片 | 项目卡片半透明模糊背景 |
| 水墨装饰线 | 技能区与价值观区的水墨风格装饰线 |

**响应式策略：**
| 断点 | 端 | 布局 |
|------|----|------|
| < 640px | 手机 | 单列，紧凑间距，小字号 |
| ≥ 640px | 平板 | 双列卡片，适中间距 |
| ≥ 1024px | 桌面 | auto-fill 自适应多列，完整尺寸 |

## 子项目体系

每个子项目是一个独立的前端应用，统一采用 Vue 3 + TypeScript 开发。

**立项到上线的完整流程：**

### 阶段一：方案设计（每次动工前）

1. 讨论确定子项目功能范围、技术选型、数据模型
2. 输出方案文档至 `{子项目}/PLAN.md`
3. 用户确认方案后进入开发阶段

### 阶段二：开发部署

1. 按方案文档实施开发
2. 完成子项目开发与部署
3. 在 `Portfolio/src/data/projects.json` 中添加一条项目卡片记录
4. 卡片中的 `url` 指向子项目的部署地址

详见 [RELEASE.md](./RELEASE.md) 的「子项目立项流程」和「新增子项目补充检查项」。

## 端口规则

| 类型 | 范围 | 用途 |
|------|------|------|
| 前端页面 | **8000–8049** | Portfolio / Vue 子项目开发端口 |
| 子项目 API | **8050–8099** | 子项目对应的 API 服务 |
| 基础设施服务 | **8100–8149** | Auth 鉴权 / 通知 / 任务调度 / 消息队列等 |

详见 [AGENTS.md](./AGENTS.md) 的端口分配表。

## 分支策略

```
main ───●────────●────────●────────  ← 始终可部署
         \      /        /
project/   ●──●──●      /
todo-app        \      /
                 ●────●

Tag: v1.0.0    v1.1.0    v1.1.1
```

- **main** — 主分支，始终可部署
- **project/\<name\>** — 子项目开发分支，长期存在的持续迭代分支；开发里程碑合并回 main 后保留，继续开发
- **feat/\<描述\>** / **fix/\<描述\>** — 小功能 / 修复
- 语义化版本：`v主版本.次版本.修订`

详见 [STYLE_GUIDE.md](./STYLE_GUIDE.md) 的 Git 章节。

## 部署架构

```
公网（HTTPS）
  │
  ▼
┌──────────────────────────────────────┐
│  Nginx (主机)                         │
│  • SSL termination (acme.sh 自动续期)  │
│  • 域名路由: lujiesheng.cn / <name>.lujiesheng.cn / api-<name>.lujiesheng.cn │
└────┬────────┬──────────┬───────────────┬────────────────┘
     │        │          │               │
     ▼        ▼          ▼               ▼
┌─────────┐ ┌─────────┐ ┌───────────────────┐ ┌──────────────────┐
│ Portfolio│ │ Monitor │ │   Monitor API     │ │  Infrastructure  │
│ :8000   │ │ :8001   │ │   :8051           │ │   Services       │
│ Docker  │ │ Docker  │ │   Docker          │ │  ┌─────────────┐ │
└─────────┘ └────┬─────┘ │ ┌──────────────┐ │ │  │ Auth 服务    │ │
                 │       │ │ Agent (Worker)│ │ │  │ :8100       │ │
                 │       │ └──────────────┘ │ │  ├─────────────┤ │
                 │       └────────┬─────────┘ │  │ Notification│ │
                 ▼                ▼           │  │  :8110       │ │
          ┌───────────────────────────┐      │  ├─────────────┤ │
          │  基础设施层 (全局共享)      │      │  │ 任务调度     │ │
          │  MySQL :3306             │      │  │ :8120       │ │
          │  Redis :6379             │      │  ├─────────────┤ │
          │  MongoDB :27017          │      │  │ 消息队列     │ │
          │  (仅 Docker 内网互联)      │      │  │ :8130       │ │
          └───────────────────────────┘      │  └─────────────┘ │
                                             └──────────────────┘
```

| 层级 | 说明 |
|------|------|
| Nginx | 主机 80/443，SSL termination，按域名反代到容器 |
| 应用容器 | 仅绑定 `127.0.0.1`，不对外暴露端口 |
| 基础设施服务 | Auth 鉴权 / 通知 / 任务调度 / 消息队列等，全局复用，`8100–8149` |
| 采集代理 | .NET 10 Background Worker，内置于 API 容器中，采集服务器/Docker/站点指标 |
| 基础设施 (数据层) | MySQL / Redis / MongoDB，所有子项目共用，仅 Docker 内网互联 |
| SSL | acme.sh + Let's Encrypt + 腾讯云 DNS API，一个域名一张免费证书 |
| CI/CD | GitHub Actions 推送到 main → SCP → docker compose up --build |

详见 [DEPLOY.md](./DEPLOY.md)

## 技术选型

| 模块 | 方案 | 理由 |
|------|------|------|
| 主站框架 | **Astro 6** | 内容型站点，默认零 JS，构建快 |
| 样式方案 | **原生 CSS + CSS 变量** | 极简无依赖，全局主题统一管理 |
| 子项目前端 | **Vue 3 + TypeScript** | 交互复杂度高时选 SPA 方案 |
| 子项目后端 | **.NET 10 + Minimal API** | 子项目 API 统一技术栈，配套 SqlSugar ORM |
| 基础设施服务 | **.NET 10 + Minimal API** | Auth 鉴权 / 通知 / 任务调度 / 消息队列等全局复用服务 |
| API 文档 | **Scalar** | 子项目 API 文档，蓝白主题适配 |
| 主站数据 | **JSON 文件** | Portfolio 数据源，简单直接
