# 子项目立项流程

在对话中说 **"规划子项目"** 或调用 `/project-init` 即可触发。

## 流程

```
有新子项目想法 → 说"规划子项目"
                     │
                     ▼
           ┌──────────────────────┐
           │ ① 讨论方案           │ ← 功能范围、技术选型、数据模型
           └────────┬─────────────┘
                     │
                     ▼
           ┌──────────────────────┐
           │ ② 输出方案文档        │ ← 写入 {子项目}/PLAN.md
           └────────┬─────────────┘
                     │
                     ▼
           ┌──────────────────────┐
           │ ③ 用户确认           │ ← 审阅方案文档，确认后锁定
           └────────┬─────────────┘
                     │
                     ▼
           ┌──────────────────────┐
           │ ④ 更新项目文档        │ ← AGENTS 子项目表 / ARCHITECTURE / DEPLOY 端口表
           └────────┬─────────────┘
                     │  (确认完毕)
                     ▼
           ┌──────────────────────┐
           │ ⑤ 创建项目分支        │ ← git checkout -b project/{name}
           └────────┬─────────────┘
                     │
                     ▼
              开始编码 → 开发完成后走 /project-release
```

## 详细步骤

### ① 讨论方案

与用户讨论以下内容，形成方案雏形：

1. **功能范围** — 子项目要做什么，核心功能列表
2. **技术选型** — 前端/后端框架、数据库、第三方服务
3. **数据模型** — 核心实体和关系
4. **项目位置** — 在 monorepo 中的目录名（PascalCase）
5. **端口分配** — 前端端口（8000-8049）+ API 端口（8050-8099）
6. **域名** — `<name>.lujiesheng.cn` + `api-<name>.lujiesheng.cn`

### ② 输出方案文档

写入 `{子项目}/PLAN.md`，包含：

```markdown
# {子项目} 方案设计

> 方案版本：v1
> 最后更新：{日期}

## 1. 项目概述

## 2. 功能范围

## 3. 技术选型

## 4. 架构设计

## 5. 数据模型

## 6. API 设计

## 7. 前端页面（如有）

## 8. 部署方案

## 9. 开发计划
```

### ③ 用户确认

输出方案文档内容摘要，请用户审阅确认。

- 用户确认后 → 进入步骤④
- 用户提出修改 → 更新 PLAN.md → 再次确认

### ④ 更新项目文档

确认后，更新以下文档（逐项检查）：

| # | 文档/操作 | 更新内容 |
|---|----------|---------|
| 1 | `Portfolio/src/data/projects.json` | 新增一个项目卡片记录 |
| 2 | `AGENTS.md` | 子项目表新增一行（端口分配表、技术栈表、目录结构） |
| 3 | `ARCHITECTURE.md` | 架构图 + 分层描述 + 部署拓扑 |
| 4 | `DEPLOY.md` + `docker-compose.yml` | 追加 docker-compose service（端口绑 `127.0.0.1`） |
| 5 | `README.md`（根） | 目录结构图 + 子项目表 |
| 6 | `CLAUDE.md`（根） | 端口分配表 + 关键决策记录 + 当前工作状态 |
| 7 | 子项目 `README.md` | 技术栈/结构/开发说明 |
| 8 | 子项目 `CHANGELOG.md` | 从 `v0.1.0` 开始 |
| 9 | 子项目 `PLAN.md` | 方案文档（步骤②已输出，确认后锁定） |
| 10 | `.github/workflows/deploy.yml` | 追加新 service 构建命令 |
| 11 | 🖐️ **DNS 解析**（手动） | 腾讯云 DNS 添加 A 记录 → `81.71.136.3` |
| 12 | 🖐️ **SSL 证书**（手动） | 服务器 acme.sh 申请独立子域名证书 |
| 13 | 🖐️ **Nginx 配置**（手动） | 追加 server 块，反代 → `127.0.0.1:<port>` |

**敏感信息红线**：所有文档中不得出现明文密码、Token、连接串含密码、API Secret 等，一律使用 `***` 或环境变量引用。

### 完成

向用户输出摘要：

```
✅ 子项目「{名称}」立项完成

方案文档：{子项目}/PLAN.md
技术文档：{子项目}/README.md
版本日志：{子项目}/CHANGELOG.md（起始 v0.1.0）

端口分配
  ├── 前端：{port} → {name}.lujiesheng.cn
  └── API：{port} → api-{name}.lujiesheng.cn

已更新文档
  ├── projects.json ✅
  ├── AGENTS.md ✅
  ├── ARCHITECTURE.md ✅
  ├── DEPLOY.md + docker-compose.yml ✅
  ├── README.md ✅
  ├── CLAUDE.md ✅
  ├── 子项目 README.md ✅
  ├── 子项目 CHANGELOG.md ✅
  ├── 子项目 PLAN.md ✅
  ├── deploy.yml ✅
  ├── DNS 解析 ✅
  ├── SSL 证书 ✅
  └── Nginx 配置 ✅

现在可以说"开始开发"进入编码阶段。
开发完成后走 /project-release 发布。
```

### ⑤ 创建项目分支

基于 main 创建子项目开发分支 `project/{子项目}`，后续开发在此分支上进行：

```bash
git checkout -b project/{子项目} main
git push origin project/{子项目}
```
